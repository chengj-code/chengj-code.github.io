Vue2源码学习

Created2025-05-17

Vue2源码解读顺序

  1. 从入口开始：先理解 new Vue() 的实例化过程，掌握生命周期和初始化流程。
  2. 深入响应式：研究 Observer、Dep、Watcher 的协作机制，理解数据绑定原理。
  3. 模板编译：分析模板如何转换为 render 函数，了解 AST 优化策略。
  4. 虚拟 DOM：学习 VNode 结构与 Diff 算法，掌握高效 DOM 更新的实现。
  5. 扩展功能：最后阅读组件系统、插件机制等高级特性，完善对框架整体的理解。

Vue 实例初始化（核心入口）

  • 关键文件：src/core/instance/index.js、src/core/instance/init.js
  • 核心逻辑：
    • 构造函数：通过 new Vue() 创建实例时，调用 this._init(options) 初始化配置项，合并选项（如 data、methods、生命周期钩子等）28。
    • Mixin 扩展：通过 initMixin、stateMixin 等函数在原型链上挂载 $data、$watch、$set 等实例方法28。
    • 生命周期初始化：依次调用 initLifecycle、initEvents、initRender 等函数，触发 beforeCreate 和 created 钩子38。

响应式系统（数据驱动核心）

  • 关键文件：core/observer 目录下的 observer.js、dep.js、watcher.js
  • 核心逻辑：
    • 数据劫持：使用 Object.defineProperty 递归劫持 data 对象的属性，为每个属性创建 Dep（依赖收集器）45。
    • 依赖收集：在 getter 中通过 Dep.target 收集当前 Watcher，建立数据与视图的关联8。
    • 派发更新：在 setter 中触发 Dep.notify()，通知所有 Watcher 执行 update 方法，触发组件重新渲染48。

模板编译与渲染函数生成

  • 关键文件：compiler 目录下的解析器、优化器、生成器模块
  • 核心逻辑：
    • 解析模板：将模板字符串转换为 AST（抽象语法树），处理指令（如 v-if、v-for）和插值表达式14。
    • 优化 AST：标记静态节点（如纯文本节点），减少后续虚拟 DOM 的对比开销47。
    • 生成渲染函数：将 AST 转换为可执行的 render 函数，用于后续生成虚拟 DOM34。

虚拟 DOM 与 DOM 更新机制

  • 关键文件：core/vdom 目录下的 vnode.js、patch.js
  • 核心逻辑：
    • 虚拟 DOM 创建：通过 render 函数生成描述 DOM 结构的虚拟节点（VNode）35。
    • Diff 算法：对比新旧 VNode，通过 patch 函数计算最小化 DOM 操作，更新实际 DOM48。
    • 异步更新队列：利用 $nextTick 将 DOM 更新操作推入微任务队列，确保数据变更后视图批量更新68。

组件系统与插件机制

  • 关键文件：core/global-api 目录下的组件注册逻辑
  • 核心逻辑：
    • 组件注册：通过 Vue.component() 注册全局组件，合并选项并生成子类构造函数45。
    • 生命周期与通信：父子组件通过 props 和 $emit 传递数据，生命周期钩子控制组件状态24。
    • 插件扩展：通过 Vue.use() 调用插件的 install 方法，扩展全局 API 或混入功能45。

其他核心模块

  • 事件系统：eventsMixin 实现 $on、$emit 等事件监听与触发机制28。
  • 服务端渲染（SSR）：server 目录下的逻辑处理服务端 HTML 生成与客户端激活1。
  • 过渡动画：通过 transition 组件管理 CSS 类名，实现元素插入/移除的动画效果5。
