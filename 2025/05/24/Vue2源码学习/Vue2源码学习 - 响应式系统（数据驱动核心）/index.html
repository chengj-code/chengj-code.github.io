Vue2源码解读顺序

  1. 从入口开始：先理解 new Vue() 的实例化过程，掌握生命周期和初始化流程。
  2. 深入响应式：研究 Observer、Dep、Watcher 的协作机制，理解数据绑定原理。
  3. 模板编译：分析模板如何转换为 render 函数，了解 AST 优化策略。
  4. 虚拟 DOM：学习 VNode 结构与 Diff 算法，掌握高效 DOM 更新的实现。
  5. 扩展功能：最后阅读组件系统、插件机制等高级特性，完善对框架整体的理解。

响应式系统（数据驱动核心）

  • 关键文件：core/observer 目录下的 index.js、dep.js、watcher.js
  • 核心逻辑：
    • 数据劫持：使用 Object.defineProperty 递归劫持 data 对象的属性，为每个属性创建 Dep（依赖收集器）。
    • 依赖收集：在 getter 中通过 Dep.target 收集当前 Watcher，建立数据与视图的关联。
    • 派发更新：在 setter 中触发 Dep.notify()，通知所有 Watcher 执行 update 方法，触发组件重新渲染。

dep.js

核心作用：实现观察者模式中的发布者角色，管理数据变更时的订阅者通知机制。每个响应式属性对应一个 Dep 实例，负责：

  1. 收集依赖（Watcher）
  2. 存储订阅者
  3. 数据变更时通知所有订阅者
  4. 提供调试支持
  5. 优化大规模订阅场景的性能

通过 pushTarget/popTarget 维护当前正在计算的依赖目标上下文，配合 Vue 的响应式系统实现精确的依赖追踪和高效的更新派发。

watcher.js

Watcher 是 Vue 响应式系统的核心观察者，负责：

  1. 依赖收集：
    • 通过执行 getter 触发属性访问
    • 收集相关 Dep 到 deps 数组
  2. 更新调度：
    • 根据配置选择同步/异步更新
    • 通过队列批量处理更新（queueWatcher）
  3. 值变更检测：
    • 对比新旧值（引用对比+深度检测）
    • 触发用户回调或重新渲染
  4. 生命周期管理：
    • 自动清理无效依赖
    • 提供销毁机制

lazy属性

lazy 属性是 Vue 响应式系统实现高效计算属性的核心机制，通过：

  1. 延迟求值：避免初始化时的冗余计算
  2. 脏检查模式：最小化重复计算次数（同一Watcher在同一事件循环中只会被加入队列一次）。
  3. 按需依赖收集：只有实际使用时才建立响应式关联
特性 普通 Watcher (lazy: false) 计算属性 Watcher (lazy: true)
初始求值 立即执行 延迟到首次访问时
依赖更新响应 立即执行回调或重新计算 仅标记为脏状态
结果缓存 无（每次更新都重新运行） 有（依赖未变时返回缓存）
典型用途 watch 选项、DOM 更新 计算属性
