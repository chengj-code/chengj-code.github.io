Vue2源码解读顺序

  1. 从入口开始：先理解 new Vue() 的实例化过程，掌握生命周期和初始化流程。
  2. 深入响应式：研究 Observer、Dep、Watcher 的协作机制，理解数据绑定原理。
  3. 模板编译：分析模板如何转换为 render 函数，了解 AST 优化策略。
  4. 虚拟 DOM：学习 VNode 结构与 Diff 算法，掌握高效 DOM 更新的实现。
  5. 扩展功能：最后阅读组件系统、插件机制等高级特性，完善对框架整体的理解。

模板编译与渲染函数生成

  • 关键文件：compiler 目录下的解析器、优化器、生成器模块
  • 核心逻辑：
    • 解析模板：将模板字符串转换为 AST（抽象语法树），处理指令（如 v-if、v-for）和插值表达式。
    • 优化 AST：标记静态节点（如纯文本节点），减少后续虚拟 DOM 的对比开销。
    • 生成渲染函数：将 AST 转换为可执行的 render 函数，用于后续生成虚拟

学习路线

Vue.js 的模板编译过程主要分布在编译器的核心模块中，涉及多个文件协同完成从模板字符串到渲染函数的转换。以下是关键文件及其作用的详细说明，按编译阶段组织：

编译入口文件

  • 流程：检测用户是否提供render函数，若无则获取template模板，调用compileToFunctions生成渲染函数179。

编译器核心流程

位于src/compiler/目录，分为三个阶段：

解析阶段（Parse → AST）

  • src/compiler/parser/index.js
    • 实现parse()函数，调用parseHTML解析模板字符串。
    • 生成AST（抽象语法树），描述模板的节点结构（如标签名、属性、子节点）。
  • src/compiler/parser/html-parser.js
    • 核心词法分析器，使用正则匹配标签、属性、文本等，通过钩子函数（start、end、chars）构建AST。

优化阶段（Optimize → 标记静态节点）

  • src/compiler/optimizer.js
    • 遍历AST，标记静态节点（static）和静态根节点（staticRoot）。
    • 目的：减少虚拟DOM比对开销，跳过静态内容的更新。

代码生成阶段（Generate → 渲染函数字符串）

  • src/compiler/codegen/index.js
    • 实现generate()和genElement()，递归AST生成渲染函数的字符串代码（如_c('div', [...]）。
    • 处理指令（如v-if、v-for）和组件。

编译器基础设施

  1. src/compiler/index.js
    • 导出createCompiler，整合基础编译流程（parse、optimize、generate）。
  2. src/compiler/to-function.js
    • 实现createCompileToFunctionFn()，将渲染字符串转换为可执行函数（通过new Function()），并添加缓存机制。
  3. src/platforms/web/compiler/options.js
    • 定义平台相关配置（如isReservedTag判断HTML标签），供编译器使用。

运行时渲染辅助工具

  • src/core/instance/render-helpers/index.js
    • 定义渲染函数中的内部工具方法（如_c（createElement）、_v（createTextVNode）、_s（toString））。
  • src/core/instance/render.js
    • 挂载_c和$createElement方法，用于创建虚拟节点。

总结：关键文件与阶段对应表

编译阶段 核心文件 主要功能
入口 entry-runtime-with-compiler.js 触发编译流程，调用compileToFunctions
解析 → AST parser/index.js, parser/html-parser.js 模板字符串 → 抽象语法树（AST）
优化 optimizer.js 标记静态节点以优化渲染性能
代码生成 codegen/index.js AST → 渲染函数字符串（如_c(...)）
函数生成 to-function.js 字符串 → 可执行渲染函数（new Function）
平台配置 platforms/web/compiler/options.js 提供Web平台相关编译选项
运行时工具 render-helpers/index.js, render.js 提供渲染函数内部工具方法（_c, _v等）

💡 完整流程：模板 → parse()生成AST → optimize()标记静态节点 → generate()生成代码字符串 → createFunction()转换为渲染函数 → 用于虚拟DOM渲染

runtime-with-compiler.js

  • 作用：完整版 Vue 的入口，重写了 $mount 方法，触发模板编译流程。

compileToFunctions

  • render：动态渲染函数
  • staticRenderFns：静态节点渲染函数
